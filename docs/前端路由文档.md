# 前端路由文档

本文档列出前端所有可用的路由路径，供后端在返回 `route_path` 字段时参考。

## 路径格式说明

- **绝对路径**：所有路径必须以 `/` 开头
- **路径参数**：使用 `:paramName` 格式，如 `/product/:id`
- **查询参数**：使用 `?key=value` 格式，如 `/search?keyword=xxx`

## 八、路径参数说明

### 动态参数格式

路径中的动态参数使用 `:paramName` 格式，后端返回时需要替换为实际值：

- **示例1**：`/product/:id` → `/product/123`
- **示例2**：`/news/:id` → `/news/456`
- **示例3**：`/cashier/:orderId` → `/cashier/ORDER20240101001`

### 查询参数格式

如需传递查询参数，使用 `?key=value` 格式：

- **示例1**：`/search?keyword=大米`
- **示例2**：`/search?keyword=大米&category=shop`

## 九、首页配置接口使用示例

### 快捷入口配置示例

```json
{
  "quick_actions": [
    {
      "id": "balance-recharge",
      "title": "专项金申购",
      "route_path": "/balance-recharge",
      "is_visible": true
    },
    {
      "id": "balance-withdraw",
      "title": "收益提现",
      "route_path": "/balance-withdraw",
      "is_visible": true
    }
  ]
}
```

## 十、注意事项

1. **路径格式**：所有路径必须以 `/` 开头，使用绝对路径
2. **参数替换**：动态参数（`:id`）需要替换为实际值
3. **路径有效性**：确保返回的路径在前端路由中存在
4. **查询参数**：如需传递查询参数，直接拼接在路径后，如 `/search?keyword=xxx`
5. **404 处理**：无效路径会自动重定向到首页 `/`
